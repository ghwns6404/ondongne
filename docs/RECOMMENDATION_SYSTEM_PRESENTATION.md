# 사용자 맞춤 추천 시스템 - 발표 자료

---

## 슬라이드 1: 개요

### 🎯 프로젝트 목표
**사용자 행동 기반 개인화 추천 시스템 구축**

- 사용자가 좋아요/조회한 상품 분석
- 카테고리별 선호도 파악
- 맞춤형 상품 추천 제공

---

## 슬라이드 2: 문제 정의

### 기존 문제점
- ❌ 모든 사용자에게 동일한 상품 표시
- ❌ 사용자 취향 무시
- ❌ 개인화 부재

### 해결 방안
- ✅ 사용자 행동 데이터 수집
- ✅ 선호도 분석 알고리즘
- ✅ 맞춤 추천 시스템

---

## 슬라이드 3: 시스템 아키텍처

```
사용자 행동
    ↓
[좋아요] [조회]
    ↓
선호도 분석
    ↓
추천 생성
    ↓
UI 표시
```

**기술 스택:**
- Flutter (Frontend)
- Firebase Firestore (Backend)
- Dart (Logic)

---

## 슬라이드 4: 알고리즘 핵심

### 1단계: 선호도 분석
```
좋아요/조회한 상품
    ↓
카테고리별 그룹화
    ↓
평균 가격 계산
    ↓
빈도순 정렬
```

### 2단계: 추천 생성
```
선호 카테고리 검색
    ↓
필터링 (가격, 중복)
    ↓
인기도 점수 계산
    ↓
정렬 & 선택
```

---

## 슬라이드 5: 인기도 점수 공식

### 공식
```
점수 = (조회수 × 2) + (좋아요 × 5)
```

### 예시
| 상품 | 조회수 | 좋아요 | 점수 |
|------|--------|--------|------|
| A | 100 | 20 | 300점 |
| B | 150 | 10 | 350점 |
| C | 80 | 30 | 310점 |

**결과:** B > C > A 순으로 추천

---

## 슬라이드 6: 필터링 전략

### 필터링 조건
1. ✅ **가격대 필터**: 평균 가격 ±50%
2. ✅ **중복 제거**: 이미 본/좋아요한 상품 제외
3. ✅ **자기 상품 제외**: 내가 등록한 상품 제외
4. ✅ **판매중만**: status = 'available'

### 예시
```
선호: 노트북, 평균 55만원
허용 범위: 27.5만원 ~ 82.5만원

✅ 50만원 노트북 → 포함
✅ 60만원 노트북 → 포함
❌ 100만원 노트북 → 제외
```

## 슬라이드 7: 데이터 모델

### Product 확장 필드
```dart
- viewCount: int              // 조회수
- viewedUserIds: List<String> // 조회한 사용자
- favoriteUserIds: List<String> // 좋아요한 사용자
- category: String            // 카테고리
```

### 데이터 수집
- **자동**: 상품 상세 진입 시 조회수 증가
- **수동**: 사용자가 좋아요 클릭

---

## 슬라이드 8: 사용자 시나리오

### 시나리오: 게이머 사용자

**1단계: 활동**
- 좋아요: 게임기 3개
- 조회: 게임기 2개, 헤드셋 1개

**2단계: 분석**
- 선호 카테고리: 게임기 (5개, 평균 35만원)

**3단계: 추천**
- 게임기 카테고리 상품 추천
- 가격 범위: 17.5만원 ~ 52.5만원
- 인기도 순 정렬

**4단계: 결과**
- 맞춤 게임기 상품 5개 추천 ✨
